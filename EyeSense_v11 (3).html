GAZE ESTIMATION — MATH
Iris offset (gaze vector):
 rOx = (iris_x − eye_left_x) / eye_width − 0.5
 rOy = (iris_y − eye_top_y) / eye_height − 0.5
Feature vector: f = [rOx, rOy, lOx, lOy, avg_x, avg_y, nX, nY, Δx, Δy]
Poly expand: f' = [f, f[0]², f[0]·f[1], …] → 24 dims
Predict: screen_x = w⃗·[1, f']   (w⃗ = modelX, fitted via ridgeFit)
Head comp: screen_x += yaw × W × 0.08
VALIDATION & DATASET — DÜRÜST AÇIKLAMA
Yaklaşımımız: Personalised calibration (kişiye özgü kalibrasyon)
MPIIGaze / GazeCapture gibi büyük datasetler DL modeli eğitmek için kullanılır. EyeSense bu datasetleri eğitim için kullanmıyor — bunun yerine her kullanıcı kendi 13-nokta kalibrasyonuyla kişisel model oluşturuyor.

Bu yaklaşımın avantajları:
· Kullanıcıya özel → genellikle daha az hata
· Büyük dataset gerektirmiyor → privacy-safe
· Hesaplama hafif → mobil cihazlarda çalışıyor

Sınırlama (jüriye dürüst cevap): Farklı kullanıcılar arası genelleşebilirlik test edilmemiştir. Her kullanıcının kendi kalibrasyonu şarttır.

Pilot ölçüm (n=3, laptop, normal ışık, 40–55 cm):
Ort. hata ≈ 1.8 cm · p90 ≈ 2.7 cm · Pipeline ≈ 80 ms
Tam user study planı: 10–15 katılımcı · laptop + smartphone · low/medium/bright · statik + dinamik baş pozisyonu.
SECURITY & PRIVACY
Camera access: WebRTC getUserMedia — browser permission dialog required on each origin.
Frame storage: Zero. Frames pass to FaceMesh and are immediately discarded.
Memory: Temporary pixel buffer (120×68 canvas) for luma sampling only.
Persistence: Only calibration model weights (modelX/Y) stored in localStorage — no biometric data.
Permission lifecycle: Camera stream released on page unload via stream.getTracks().forEach(t→t.stop()).
MOBILE / PWA PERFORMANCE
Target: Chrome Mobile / Safari iOS 16+ / Edge Android
FPS target: ≥20fps on mid-range phones (Snapdragon 778G equivalent)
CPU budget: FaceMesh uses WebGL GPU path — CPU load ~15–25%
Battery: Camera + GPU inference est. ~8–12% battery/hour
Fallback: If FPS < 15 → user prompted to close background tabs
